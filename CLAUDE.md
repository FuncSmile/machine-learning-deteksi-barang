## What this is

A YOLOv8 (Ultralytics) object-detection project for 26 Indonesian household-item classes
(`HP`, `botol`, `gelas`, `laptop`, `mouse`, `dompet`, etc. â€” full list in `datasets/data.yaml`).
The dataset is a Roboflow export (`deteksi-barang-qhjji`, v2, ~2569 images). Code comments,
print output, and docstrings are written in Indonesian â€” match that language when editing them.
The stated quality target is **per-class Precision â‰Ą 0.90**, which threads through `train.py`
and `evaluate.py`.

## Environment

- The virtualenv lives in `OBJECT_DETECTION/` (Python 3.14). Activate it before running anything:
  `source OBJECT_DETECTION/bin/activate`. Inside it the interpreter is `python`; outside, the
  system Python is `python3` and is PEP 668 "externally-managed" (a bare `pip install` is rejected â€”
  a venv is mandatory). See `keep.txt` for the original setup notes.
- Install deps: `pip install -r requirements.txt`. For NVIDIA GPUs, install a CUDA build of
  `torch`/`torchvision` first (see the comment block at the top of `requirements.txt`); otherwise
  training runs on CPU and is very slow for `yolov8m` Ă— 150 epochs.
- Not a git repository. There is no test suite, linter, or build step â€” "running" means executing
  the pipeline scripts below.

## The pipeline (run in this order, from the project root)

```bash
python fix_yaml.py        # rewrite data.yaml split paths to absolute (idempotent)
python check_dataset.py   # validate dataset before spending GPU time
python train.py           # train -> runs/train/deteksi_barang_v1/weights/best.pt
python evaluate.py        # evaluate best.pt on the TEST split
python predict.py --mode webcam    # live inference (also: --mode video|image --source PATH)
```

Each script is standalone (`if __name__ == "__main__"`), takes no config flags except
`predict.py`, and exits non-zero on failure. `predict.py` flags: `--mode webcam|video|image`,
`--source PATH` (required for video/image), `--camera N` (webcam index, default 0); press `q` to
quit a stream.

## Architecture and cross-file contracts

The scripts are decoupled but coordinate through **hardcoded path and name conventions**, not
through a shared config module. If you change one, grep for the others:

- **`RUN_NAME = "deteksi_barang_v1"`** and the derived path
  `runs/train/deteksi_barang_v1/weights/best.pt` are duplicated as constants in `train.py` (writes
  it), `predict.py`, and `evaluate.py` (both read it). Renaming the run requires editing all three.
- **`datasets/data.yaml`** is the single source of truth for class `names`/`nc`. Every script loads
  it via `yaml.safe_load` and normalizes `names` (handles both list and `{id: name}` dict forms).
  Its `train`/`val`/`test` values are **absolute paths specific to this machine**, regenerated by
  `fix_yaml.py` â€” do not hand-commit them as portable; on a new checkout run `fix_yaml.py` first.
  `train.py` auto-invokes `fix_yaml.fix_yaml()` at startup (imports it as a module).
- Heavy imports (`ultralytics`, `torch`, `cv2`) are deliberately done **inside functions**, not at
  module top level, so that path-fixing / clear error messages still work when packages are missing.
  Preserve this pattern.
- Outputs land under `runs/`: `runs/train/<name>/` (weights, `results.csv`, plots),
  `runs/evaluate/` (copied confusion matrices), `runs/predict/` (`output_video.mp4`,
  `output_image.jpg`). Top-level reports `results_summary.txt` and `evaluation_report.txt` are also
  written.

### Key behaviors worth knowing before editing

- **`train.py`** holds all hyperparameters in the `TRAIN_CONFIG` dict (model `yolov8m.pt`, 150
  epochs, imgsz 640, AdamW, augmentation knobs). On CUDA OOM it automatically retries with smaller
  batches via `BATCH_FALLBACK = [16, 8, 4]`, clearing the GPU cache between attempts; OOM detection
  is string-matching in `is_oom_error()`. Device is auto-detected (`detect_device()` â†’ `"cuda"`/`"cpu"`).
- **`evaluate.py`** evaluates on `split="test"`, builds a per-class P/R/F1/mAP table, flags classes
  below `PRECISION_TARGET = 0.90`, and lists classes with no test predictions. It tolerates
  Ultralytics version differences via a try/except fallback around `box.class_result(i)`.
- **`predict.py`** uses `CONF_THRESHOLD = 0.15` / `IOU_THRESHOLD = 0.45` (tuned low for recall) and
  draws its own OpenCV overlay (boxes, per-class counts, FPS). `imshow` is wrapped so it degrades
  gracefully on headless machines.
- **`check_dataset.py`** intentionally depends only on the stdlib + `pyyaml` (no ultralytics/torch),
  so it can run before the heavy deps are installed. It warns on classes with `< 30` instances and
  on orphan image/label files.
